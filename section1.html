시멘틱 태그란 무엇인가?


시멘틱(Semantic) 태그는 이름에서 알 수 있듯이 태그의 의미가 명확히 드러나는 HTML 태그를 말합니다.
이 태그를 사용하면 웹 페이지의 구조와 내용이 더욱 이해하기 쉽게 됩니다.
브라우저와 검색 엔진, 그리고 화면 읽기 장치와 같은 보조 기술도 이러한 태그를 통해 페이지의 구조를 파악할 수 있습니다.



시멘틱 태그의 중요성


  • SEO(검색 엔진 최적화): 검색 엔진은 시멘틱 태그를 통해 페이지의 콘텐츠를 더 잘 이해하고 검색 순위를 매길 수 있습니다.

  • 접근성: 화면 읽기 장치나 보조 기술이 콘텐츠를 보다 정확히 전달할 수 있습니다.

  • 가독성 및 유지보수성: 코드를 읽는 개발자가 페이지 구조를 더 쉽게 이해할 수 있습니다.


자주 사용되는 시멘틱 태그


1. <header>


페이지나 섹션의 머리말 역할을 합니다. 일반적으로 로고, 내비게이션, 제목 등이 포함됩니다.


                <header>
                  <h1>웹디자인 기능사</h1>
                  <nav>
                    <ul>
                      <li><a href="#home">홈</a></li>
                      <li><a href="#about">소개</a></li>
                      <li><a href="#contact">문의</a></li>
                    </ul>
                  </nav>
                </header>

2. <nav>


내비게이션 링크를 그룹화하는 데 사용됩니다. 페이지 내 또는 외부 링크를 포함할 수 있습니다.


                <nav>
                  <ul>
                    <li><a href="#">Home</a></li>
                    <li><a href="#">Services</a></li>
                    <li><a href="#">Contact</a></li>
                  </ul>
                </nav>

3. <main>


문서의 주요 콘텐츠를 감싸는 태그입니다. 한 문서에 한 번만 사용해야 합니다.

<main>
                  <article>
                    <h2>시멘틱 태그 배우기</h2>
                    <p>시멘틱 태그는 웹 접근성과 SEO에 매우 중요합니다.</p>
                  </article>
                </main>

4. <article>


독립적으로 구분 가능한 콘텐츠 블록을 나타냅니다. 블로그 글, 뉴스 기사 등에 적합합니다.

<article>
                  <h2>HTML5란?</h2>
                  <p>HTML5는 웹의 최신 표준으로, 시멘틱 태그가 포함되어 있습니다.</p>
                </article>

5. <section>


문서의 논리적 구분을 나타냅니다. 제목과 함께 사용하여 페이지를 더 구조화할 수 있습니다.

<section>
                  <h2>웹디자인의 중요성</h2>
                  <p>좋은 웹디자인은 사용자 경험을 향상시킵니다.</p>
                </section>

6. <aside>


주요 콘텐츠와는 별도로 부가적인 정보를 제공할 때 사용합니다.

<aside>
                  <h3>참고 자료</h3>
                  <p>HTML, CSS 및 JavaScript에 대한 더 많은 정보를 보려면 공식 문서를 확인하세요.</p>
                </aside>

7. <footer>


문서나 섹션의 바닥글 역할을 합니다. 저작권 정보, 연락처, 사이트맵 등이 포함될 수 있습니다.

<footer>
                  <p>&copy; 2024 웹디자인 기능사 준비 사이트</p>
                </footer>


헷갈릴 수 있는 내용들


  • <section> vs <div>
    • <section>은 논리적 그룹화, <div>는 스타일링이나 스크립팅 목적으로 사용합니다.
    • <section>은 항상 제목(<h1>~<h6>)과 함께 사용해야 합니다.

  • <article> vs <section>
    • <article>은 독립적으로 사용할 수 있는 콘텐츠, <section>은 관련된 콘텐츠를 묶는 데 사용됩니다.

  • <header> vs <head>
    • <header>는 페이지나 섹션의 머리말, <head>는 메타데이터를 포함하는 HTML 문서의 상단에 위치합니다.

  • 한 문서에 <main>은 하나만 사용
    • <main>은 문서의 주요 콘텐츠를 나타내므로 중복 사용하지 않습니다.
